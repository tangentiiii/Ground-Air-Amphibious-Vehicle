# MaixCAM Pro 无线目标检测视频流

该程序在 MaixCAM Pro 上完成目标检测，通过局域网分别发送未画框的视频和检测元数据。浏览器使用 Canvas 在电脑端绘制红框、类别和置信度，电脑端无需安装 Python 依赖。

## 准备工作

1. 将 MaixCAM Pro 固件升级到较新的 MaixPy 4.x。YOLO11 要求 MaixPy >= 4.7.0；固件还需包含 `maix.http.JpegStreamer`。
2. 在 MaixCAM 的“设置”中连接 Wi-Fi，并让电脑连接同一个路由器/局域网。
3. 确认设备中存在模型 `/root/models/yolo11n.mud`。若没有，请从 MaixHub 模型库安装 YOLO11，或修改代码顶部的 `MODEL_PATH` 和 `MODEL_TYPE` 使用已有模型。

默认模型识别 COCO 数据集中的 80 类常见物体。若要识别赛道标志、特定车辆或自定义障碍物，需要训练自己的 YOLO 模型并转换为 `.mud` 文件。

## 摄像头竖装（逆时针旋转 90°）

程序默认按“摄像头相对正常横向姿态逆时针旋转 90°”处理：先将每帧顺时针转回 90°，再送入 YOLO 检测。程序会把模型坐标精确转换成视频流中的归一化坐标，浏览器据此绘框。

代码中的 `FRAME_ROTATION_DEGREES = 270` 使用的是 MaixPy 的逆时针角度，即逆时针 270° 等于顺时针 90°。不同安装方向对应设置如下：

| 摄像头物理安装方向 | `FRAME_ROTATION_DEGREES` |
| --- | ---: |
| 正常横向 | `0` |
| 逆时针旋转 90° | `270` |
| 旋转 180° | `180` |
| 顺时针旋转 90° | `90` |

旋转 90° 后，画面会先成为竖向比例，再使用 `FIT_CONTAIN` 保持比例缩放到模型输入尺寸；两侧出现黑边是正常现象。这比把竖向画面强制拉伸为横向更有利于识别。

## 运行

1. 用 MaixVision 连接 MaixCAM Pro。
2. 打开 `maixcam_object_stream.py`，先直接运行测试；确认无误后可安装为设备应用或配置开机运行。
3. 从设备的“设置 → Wi-Fi”查看局域网 IP，例如 `192.168.1.50`。
4. 在电脑浏览器打开 `http://192.168.1.50:8000/`。

也可以直接访问视频地址：`http://192.168.1.50:8000/stream`。

网页顶部可以实时选择 `320×240` 或 `640×480`，以及 5–30 FPS。视频使用 8000 端口；检测元数据和配置接口使用 8001 端口，因此电脑防火墙和路由器都必须允许访问这两个端口。

红框和文字由浏览器的高 DPI Canvas 绘制，不会写入 MaixCAM 回传的 JPEG。可直接打开 `http://设备IP:8000/stream` 验证原始视频：该地址不应出现任何框或文字。如果该地址仍有标注，说明设备上运行的还是旧版程序，应停止旧应用、重新上传并启动本程序，然后强制刷新浏览器页面。

## 常用配置

编辑程序顶部常量：

- `CONFIDENCE_THRESHOLD`：检测置信度阈值，误检多则提高，漏检多则降低。
- `IOU_THRESHOLD`：检测框去重阈值。
- `CAPTURE_WIDTH` / `CAPTURE_HEIGHT`：摄像头原始采集分辨率，也是回传画质上限。
- `SHOW_ON_DEVICE`：不需要设备屏幕显示时设为 `False`，可减少额外显示开销。
- `VIDEO_PORT` / `CONTROL_PORT`：视频网页端口及元数据控制端口。
- `MODEL_PATH` / `MODEL_TYPE`：模型路径和模型类型。
- `FRAME_ROTATION_DEGREES`：送入识别前的逆时针旋转角度。

视频采集、YOLO 输入和网络回传分辨率相互独立。提高网络回传分辨率会增加 JPEG 编码和无线带宽，但不会提高模型本身的输入分辨率；如需提高远处小目标识别率，应更换输入尺寸更大的 `.mud` 模型。

## 故障排查

- 浏览器打不开：确认两端在同一局域网、IP 无误，关闭路由器的 AP/客户端隔离，并检查电脑防火墙。
- 提示找不到模型：检查 `/root/models/` 中实际的 `.mud` 文件名并修改 `MODEL_PATH`。
- 提示没有 `YOLO11`：升级固件，或将 `MODEL_TYPE` 改为 `yolov8`/`yolov5` 并使用对应模型。
- 提示没有 `http.JpegStreamer`：升级到包含 `maix.http` 的新版 MaixPy 固件。
- 延迟较高：降低模型输入分辨率、关闭设备本地显示，或改善 Wi-Fi 信号。
- 竖装后远处小目标漏检：竖向画面适配横向模型时有效像素会减少；建议改用输入尺寸更大或接近正方形的 `.mud` 模型。
